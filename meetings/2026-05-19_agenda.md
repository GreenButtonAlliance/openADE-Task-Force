## NAESB ESPI Standard Changes for ESPI Version 4.1
* NAESB released the ESPI Version 4.1 standard on May 15th
  * SFTP is no longer a supported protocol
  * [Deprecated] items are now standardized and include the deprecated item description

## NAESB ESPI Standard Retail Customer Schema Simplification
* The original ESPI Retail Customer Schema is based on the Customer and Document CIM schemas
  * NAESB Request R26003 simplifies the current Retail Customer schema to match the ESPI Energy schema 
    IdentifiedObject assignment pattern
    * Review of Enterprise Architecture UML Retail Customer UML Diagrams
    * Review of ESPI Retail Customer schema
  * Next steps:
    * Generate NAESB ESPI version 4.2 standard red-line document incorporating the UML diagram and Retail Customer 
      schema changes 
  * Impact on existing ESPI Retail Customer implementations:
    * No impact on existing ESPI Retail Customer implementations as there are no data XML format changes
    * Changes should simplify DataCustodian and Third Party DB designs
